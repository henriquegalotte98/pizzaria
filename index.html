📖 Introdução

Bem-vindo ao Projeto Pizzaria

Este projeto foi criado para que os alunos aprendam como um software real é desenvolvido utilizando metodologias ágeis, divisão de tarefas, GitHub, Front-End, Back-End, banco de dados e integração entre sistemas.

Ao final do projeto, todos terão contato com:

  • Scrum
  • Git e GitHub
  • Front-End com HTML, CSS e JavaScript
  • Back-End com Spring Boot
  • Banco de Dados MySQL
  • APIs REST
  • Integração Front-End e Back-End
  • Trabalho em equipe
Importante: o objetivo não é criar uma pizzaria perfeita. O objetivo é aprender os conceitos e entregar um sistema funcional dentro de apenas 3 Sprints.

🎯 Objetivo do Projeto

Desenvolver um sistema web para uma pizzaria que permita:

  • Visualizar cardápio
  • Visualizar imagens das pizzas
  • Adicionar pizzas ao carrinho
  • Finalizar pedidos
  • Consultar pedidos pelo telefone
  • Acessar uma Área de Funcionários
  • Gerenciar pizzas no Dashboard
  • Fazer upload de imagens das pizzas
  • Visualizar pedidos
  • Alterar status dos pedidos
O sistema terá somente retirada no balcão. Não haverá entrega por motoboy, pagamento real, PIX integrado, autenticação avançada ou Spring Security.

🗄 Banco de Dados

Modelo Conceitual

Cliente possui vários pedidos
⬇ 1:N
Pedido possui vários itens
⬇ 1:N
ItemPedido representa uma pizza dentro do pedido
⬇ N:1
Produto pizza cadastrada no cardápio

DER Simplificado

CLIENTE
- id
- nome
- telefone
- dataCadastro

1 CLIENTE possui N PEDIDOS

PEDIDO
- id
- cliente_id
- dataHoraPedido
- horarioRetirada
- status
- formaPagamento
- observacao
- valorTotal

1 PEDIDO possui N ITENS_PEDIDO

ITEM_PEDIDO
- id
- pedido_id
- produto_id
- tamanho
- quantidade
- precoUnitario
- subtotal

N ITENS_PEDIDO pertencem a 1 PRODUTO

PRODUTO
- id
- nome
- descricao
- precoPequena
- precoMedia
- precoGrande
- imagem
- ativo

CLIENTES

  • id — BIGINT
  • nome — VARCHAR(100)
  • telefone — VARCHAR(20), único
  • dataCadastro — DATETIME

PRODUTOS

  • id — BIGINT
  • nome — VARCHAR(100)
  • descricao — TEXT
  • precoPequena — DECIMAL(10,2)
  • precoMedia — DECIMAL(10,2)
  • precoGrande — DECIMAL(10,2)
  • imagem — VARCHAR(255)
  • ativo — BOOLEAN

PEDIDOS

  • id — BIGINT
  • cliente_id — BIGINT
  • dataHoraPedido — DATETIME
  • horarioRetirada — TIME
  • status — VARCHAR(30)
  • formaPagamento — VARCHAR(20)
  • observacao — TEXT
  • valorTotal — DECIMAL(10,2)

ITENS_PEDIDO

  • id — BIGINT
  • pedido_id — BIGINT
  • produto_id — BIGINT
  • tamanho — CHAR(1)
  • quantidade — INT
  • precoUnitario — DECIMAL(10,2)
  • subtotal — DECIMAL(10,2)

Regras de Negócio

  • Cliente é identificado pelo telefone.
  • Telefone deve ser único.
  • Cliente não pode ser cadastrado sem nome.
  • Produto inativo não deve aparecer no cardápio.
  • Os preços P, M e G devem ser maiores que zero.
  • Um pedido deve possuir pelo menos um item.
  • O valor total deve ser calculado automaticamente.
  • O status inicial do pedido deve ser PENDENTE.
  • A observação do pedido é opcional.
  • A quantidade do item deve ser maior que zero.
  • O tamanho da pizza deve ser P, M ou G.
  • O preço do item deve ser copiado no momento do pedido.
  • Não é permitido voltar status anterior.
